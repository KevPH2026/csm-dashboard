# AIMI 客户成功价值分析系统 — 精细化呈现设计规则

## 一、业务背景与核心矛盾

| 维度 | AIMI（新业务） | 广告（老业务） | 独立站（老业务） |
|------|---------------|---------------|-----------------|
| 成熟度 | 新产品，需精细化运营数据呈现 | 成熟业务，有续费追踪体系 | 成熟业务，**无续费追踪体系** |
| 续费关注 | ✅ 核心指标 | ✅ 有在关注续费 | ❌ 不关注续费（认为续费不重要） |
| 核心诉求 | 运营清晰可见，是新增长引擎 | 稳定支撑，续费是核心 | 询盘转化/毛利是核心 |
| 呈现要求 | **主角位**，突出运营细节 | **配角位**，不干扰AIMI | **配角位**，不干扰AIMI |

### 核心设计矛盾
1. **AIMI 需要突出，但不能孤立** — 新业务需要清晰的数据呈现，但客户是跨产品的
2. **老业务不能干扰 AIMI** — 广告/独立站数据不应喧宾夺主
3. **独立站没有续费体系** — 不应强求独立站展示续费率等无意义指标
4. **客户视角是统一的** — 一个客户可能买 AIMI + 广告 + 独立站

---

## 二、核心设计原则

### 原则 1：AIMI 主角，老业务配角
- **AIMI 数据始终处于视觉主位**（更大卡片、更鲜明颜色、更靠前位置）
- 老业务（广告/独立站）数据作为**支撑上下文**，而非并列展示
- 老业务卡片视觉权重降低（更淡的背景、更小的字号、更窄的间距）

### 原则 2：客户视角统一，产品线拆解为辅助
- **主视角永远是客户**：一个客户买了什么产品、健康度如何、价值如何
- **产品线拆解是辅助视角**：AIMI/广告/独立站的分线数据是"切面"而非"首页"
- 多产品交叉分析是核心差异化，必须突出

### 原则 3：指标真实性原则 — 不造数据
- 独立站**不展示续费率**（因为实际没有追踪）
- 广告**展示续费率**（因为实际在追踪）
- 没有追踪的指标用"—"或"暂无数据"代替，不用虚假数字填充
- 每个产品只展示对其有意义的指标

### 原则 4：层次分明 — 3秒看懂 AIMI，30秒看完全局
- **3秒层**：顶部 KPI 只展示 AIMI 核心指标 + 全局客户健康度
- **30秒层**：下方模块展开多产品交叉和老业务上下文
- 不要把三条产品线指标并列放在同一层级，导致信息过载

- 底部核心洞察突出 AIMI 的交叉销售价值

## 四、顶部 KPI Strip 重构

**现状**：6 个 KPI 混合展示 AIMI 和全局指标

**重构后**：

```
┌───────────────────────────────────────────────────────────────┐
│ AIMI 核心指标（大号，突出）                                    │
│ ┌─────────┐ ┌─────────┐ ┌──────────┐ ┌──────────┐          │
│ │产品使用率│ │粉丝增长率│ │AI功能渗透│ │ 询盘率   │          │
│ │  68.5%  │ │  8.7%   │ │  62.3%   │ │  3.8%    │          │
│ └─────────┘ └─────────┘ └──────────┘ └──────────┘          │
│                                                               │
│ 全局概览（小号，灰色调）                                       │
│ 总客户237家 · 健康率30% · NPS 58 · 续费率72.4%               │
│ 老业务: 广告ROAS 3.2x · 独立站转化5.8%                       │
└───────────────────────────────────────────────────────────────┘
```

**关键变化**：
- 第一行只展示 AIMI 的 4 个核心指标
- 第二行灰色小字展示全局概览和老业务关键指标
- 不再把三条产品线指标并列

```typescript
// AIMI 运营深度数据

customerComboDetail: {
  aimiOnly: { count: number; ratio: number; arpu: number; renewalRate: number | null; ltv: number; healthAvg: string };
  aimiAds: { count: number; ratio: number; arpu: number; renewalRate: number; ltv: number; healthAvg: string };

  fullChain: { count: number; ratio: number; arpu: number; renewalRate: number; ltv: number; healthAvg: string };
  adsOnly: { count: number; ratio: number; arpu: number; renewalRate: number; ltv: number; healthAvg: string };

  siteOnly: { count: number; ratio: number; arpu: number; renewalRate: number | null; ltv: number; healthAvg: string };

```

- `productLineValue.site.renewalRate` → 移除（独立站无续费追踪）
- `renewalChurn.site.renewalRate` → 改为客户生命周期指标
- 顶部 KPI 中移除全局续约率/增购率（移到全局概览小字中）
